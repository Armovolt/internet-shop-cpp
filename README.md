# Інформаційна система «Облік Продажів»

Консольна утиліта для автоматизації обліку номенклатури товарів, ведення бази контрагентів та фіксації угод. Виконана в рамках навчальної практики. 
Розроблена на C++ з використанням графічної бібліотеки ncurses (сумісна з OnlineGDB / Linux).

## Структура бази даних

Система складається з трьох взаємопов'язаних таблиць.

### 1. Товари (Каталог)
Основна таблиця для ведення каталогу. Включає цінову політику та опис номенклатури.
* **Код товару:** Унікальний ідентифікатор (PK)
* **Найменування:** Назва продукту
* **Гуртова ціна:** Ціна для великих партій
* **Роздрібна ціна:** Ціна для кінцевого споживача
* **Опис:** Характеристики товару

### 2. Покупці (База контактів)
Таблиця для зберігання інформації про клієнтів.
* **Код покупця:** Унікальний ідентифікатор (PK)
* **Телефон:** Контактний номер
* **Адреса:** Адреса доставки/реєстрації
* **Контактна особа:** ПІБ покупця

### 3. Угоди (Продажі)
Сполучна ланка, що забезпечує цілісність зв'язків між асортиментом та клієнтською базою.
* **Transaction_ID:** Унікальний код угоди
* **Дата:** Дата та час здійснення купівлі
* **Код товару:** Посилання на куплений товар (FK)
* **Кількість:** Кількість купленого товару
* **Код покупця:** Посилання на покупця (FK)
* **Ознака гурту:** Ознака оптового продажу (1 — опт, 0 — роздріб)

## Функціонал програми
* Створення та перегляд каталогу товарів.
* Реєстрація покупців та оформлення угод.
* Збереження всіх даних у локальні файли між сесіями запуску.
